Frequency Tables for Grouped Data

Organizing and summarizing large datasets effectively.

What is a Frequency Table?

Imagine you have collected a lot of data, like the test scores of 100 students. Looking at a long list of numbers can be confusing and hard to understand. A frequency table helps us organize this data by showing how often each value, or a range of values, appears.

For grouped data, instead of listing every single score, we group scores into "classes" or "intervals." This is especially useful when dealing with continuous data or a very wide range of discrete values.

Key Components of a Grouped Frequency Table

A typical grouped frequency table includes:

  • Class Intervals: These are the ranges into which the data is grouped (e.g., 60-69, 70-79).
  • Frequency (f): The number of data points that fall within each class interval.
  • Relative Frequency: The proportion of the total data that falls into each class. It's calculated as (Class Frequency / Total Number of Data Points). Often expressed as a decimal or percentage.
  • Cumulative Frequency: The running total of frequencies. It shows how many data points are below the upper limit of a given class.
  • Class Mark (Midpoint): The middle value of a class interval. It's calculated as (Lower Limit + Upper Limit) / 2. Useful for calculations like the mean of grouped data.

How to Construct a Grouped Frequency Table (Step-by-Step)

Let's say we have a dataset of student test scores ranging from 55 to 98.

  1. Determine the Range of the Data: The range is the difference between the highest and lowest values.

    Range = Maximum Value - Minimum Value

  2. Decide on the Number of Classes (k): There's no strict rule, but usually between 5 and 20 classes. Too few can hide details, too many can be overwhelming. A common guideline is to use $k \approx \sqrt{n}$, where $n$ is the total number of data points.
  3. Calculate the Class Width (w): This determines the size of each interval. Always round up to ensure all data is covered.

    Class Width (w) = Range / Number of Classes (k)

  4. Establish Class Limits: Start with the minimum value or a convenient number slightly below it for the first lower class limit. Add the class width to find the lower limit of the next class, and so on. Make sure classes are mutually exclusive (no overlaps) and exhaustive (cover all data).
  5. Tally the Frequencies: Go through your raw data and count how many observations fall into each class interval.
  6. Calculate Relative and Cumulative Frequencies (Optional): Once frequencies are tallied, you can easily compute the relative and cumulative frequencies.

Example: Student Test Scores

Suppose we have the following 20 test scores:

55, 60, 63, 67, 70, 72, 75, 78, 80, 81, 83, 85, 87, 88, 90, 92, 95, 96, 97, 98

Let's construct a frequency table with 5 classes:

  1. Range: $98 - 55 = 43$
  2. Number of Classes (k): 5 (given)
  3. Class Width (w): $43 / 5 = 8.6 \rightarrow$ round up to 9

Now, let's create the table:

Class Interval Class Mark (x) Frequency (f) Relative Frequency Cumulative Frequency
55 - 63 59 3 $3/20 = 0.15$ 3
64 - 72 68 3 $3/20 = 0.15$ $3+3 = 6$
73 - 81 77 4 $4/20 = 0.20$ $6+4 = 10$
82 - 90 86 5 $5/20 = 0.25$ $10+5 = 15$
91 - 99 95 5 $5/20 = 0.25$ $15+5 = 20$
Total 20 1.00

From this table, we can quickly see that 25% of students scored between 82 and 90, and 75% of students scored 90 or below (from cumulative frequency). This makes large datasets much easier to understand!
